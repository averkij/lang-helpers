# Инструменты полевой лингвистики

Автоматизация обработки полевых лингвистических данных:
от глоссированных текстов к корпусам, словарям и поисковым интерфейсам.

---

## Как пользоваться

### 1. Установите Python

Требуется Python 3.11+. Скачайте с https://www.python.org/downloads/

При установке на Windows отметьте галочку **"Add Python to PATH"**.

### 2. Запустите сервер

**Windows:** дважды щёлкните по файлу `start.cmd`.

**macOS / Linux:**

```bash
chmod +x start.sh
./start.sh
```

Скрипт проверит Python, установит зависимости и запустит веб-сервер.

### 3. Откройте браузер

Перейдите по адресу: http://localhost:8000

На главной странице три модуля: **Транслятор**, **Лексикограф**, **Поиск по корпусу**.
Ниже -- список доступных языков. По умолчанию доступен нивхский (`nivkh`).

---

## Транслятор

Конвертирует глоссированные тексты в форматы JSON-корпуса и CoNLL-U.

1. Откройте Транслятор (карточка на главной или `http://localhost:8000/translator`).
2. Загрузите `.txt` файл с глоссированным текстом (перетащите или нажмите на область загрузки).
3. Выберите языковую схему в выпадающем списке.
4. Нажмите одну из кнопок:
   - «Конвертировать в JSON» -- только JSON-корпус.
   - «Конвертировать в CoNLL-U» -- только CoNLL-U.
   - «Конвертировать всё» -- оба формата.
5. Просмотрите и скачайте результаты (кнопки «Скачать» и «Копировать»).

### Как использовать результаты

- **JSON** -- загрузите в поисковый модуль системы или обработайте программно (Python, R, JavaScript).
- **CoNLL-U** -- откройте в инструментах UD (UDPipe, Stanza, Grew, Arborator), используйте для обучения NLP-моделей.

## Лексикограф

Извлекает из глоссированного текста словарные пары и формирует двуязычные словники.

1. Откройте Лексикограф (карточка на главной или `http://localhost:8000/lexicograph`).
2. Загрузите `.txt` файл с глоссированным текстом.
3. Нажмите «Создать словник».
4. Просмотрите результат: список лемм с переводами и грамматической информацией.
5. Скачайте словник в формате **JSON** или **TXT** (также доступна кнопка «Копировать»).

## Поиск по корпусу

Позволяет находить слова и конструкции в корпусе по лингвистическим критериям.

**Предварительно:** загрузите тексты через Транслятор и сконвертируйте их в JSON.

1. Перейдите на страницу поиска (карточка «Поиск по корпусу» или `http://localhost:8000/search/nivkh`). Для другого языка замените `nivkh` на его код.
2. Введите запрос в строку поиска. Режимы:
   - **Словоформа** -- поиск по точной форме слова.
   - **Лемма** -- поиск по словарной форме / корню.
   Строку можно оставить пустой и искать только по грамматическим критериям.
3. Выберите часть речи (NOUN, VERB, NUM, ADV, PRON, PROPN). Можно выбрать несколько.
4. Задайте грамматические признаки: раскройте нужные категории (падеж, число, наклонение и т.д.) и отметьте значения.
5. Критерии разных типов объединяются по **И** (AND), значения внутри одной категории -- по **ИЛИ** (OR).
6. Нажмите «Найти». Кнопка «Сбросить» очищает все поля.

## Добавление нового языка

Поддержка нового языка добавляется без написания кода -- достаточно создать языковую схему.

### Шаг 1. Скопируйте шаблон

Скопируйте `schemas/SCHEME_TEMPLATE.py` с именем по ISO 639-3 (три буквы):

```
schemas/SCHEME_TEMPLATE.py  -->  schemas/ket.py
```

Код языка: https://iso639-3.sil.org/

### Шаг 2. Заполните метаданные

```python
language_name = "Кетский"
language_name_en = "Ket"
language_code = "ket"
```

### Шаг 3. Заполните морфологический словарь

Для каждого Лейпцигского сокращения в ваших текстах добавьте запись в `morphdict`:

```python
morphdict = {
    "NOM": "Case=Nom",
    "ACC": "Case=Acc",
    "DAT": "Case=Dat",
    "SG": "Number=Sing",
    "PL": "Number=Plur",
    "1": "Person=1",
    "IND": "Mood=Ind",
    "IMP": "Mood=Imp",
    "PST": "Tense=Past",
    "NEG": "Polarity=Neg",
    # ...
}
```

Типы записей:

| Тип | Формат | Пример |
|-----|--------|--------|
| Простое соответствие | `"ГЛОССА": "Категория=Значение"` | `"ABL": "Case=Abl"` |
| Составное соответствие | `"ГЛОССА": "Кат1=Знач1\|Кат2=Знач2"` | `"3SG": "Person=3\|Number=Sing"` |
| Булевый признак | `"ГЛОССА": "Признак=Yes"` | `"FOC": "Focus=Yes"` |
| Часть речи | `"ГЛОССА": "POS_TAG"` | `"DISC": "INTJ"` |

Справочник UD-признаков: https://universaldependencies.org/u/feat/index.html

### Шаг 4. Настройте типологические параметры

```python
adjectives = True   # Есть ли прилагательные как отдельная часть речи?
prefixes = False    # Использует ли язык префиксы?
```

### Шаг 5. Задайте значения по умолчанию (необязательно)

```python
defaults = {
    "NOUN": ["Case=Nom", "Number=Sing"],
    "VERB": [],
}
```

### Шаг 6. Перезапустите сервер

Остановите сервер (`Ctrl+C`) и запустите заново. Новый язык появится во всех интерфейсах и получит страницу поиска по адресу `http://localhost:8000/search/<код_языка>`.

## Подготовка данных

Система принимает файлы `.txt` в кодировке UTF-8, подготовленные по Лейпцигским правилам глоссирования.

### Формат предложения (четыре строки)

```
N> исходный текст предложения
морфемно-сегментированный текст
глоссы-через-ДЕФИСЫ
'Свободный перевод'
```

Предложения разделяются **пустой строкой**. Дефис (`-`) -- граница аффикса, знак равенства (`=`) -- граница клитики. Лексические глоссы -- строчными, грамматические -- ПРОПИСНЫМИ.

### Пример

```
1> чʼо ытик нивғ-ғун тʼатл ӈа-й-ра
чʼо ытик нивғ-ғун тʼатл ӈа-й-ра
один один человек-ABL рассказ быть-IND-EMPH
'У одного человека был рассказ'

2> тыф ирг-ахт кеӈ-гу-д
тыф ирг-ахт кеӈ-гу-д
он медведь-охотник быть.хорошим-CAUS-IND
'Он был хорошим охотником на медведей'
```

### Частые ошибки

- Несовпадение числа слов между строками сегментации и глосс.
- Несовпадение числа морфем внутри слова.
- Грамматические глоссы строчными буквами (`abl` вместо `ABL`).
- Отсутствие пустой строки между предложениями.
- Отсутствие номера предложения (`N>`).
- Пробелы вместо дефисов внутри словоформ.

## FAQ

**Система не запускается?**
Проверьте: Python 3.11+ установлен, добавлен в PATH, зависимости установлены (`pip install -r requirements.txt`), порт 8000 свободен.

**Как остановить сервер?**
`Ctrl+C` в терминале.

**Как изменить порт?**
В файле `start.cmd` / `start.sh` замените `--port 8000` на нужный номер.

**Работает ли без интернета?**
Да, после установки зависимостей система полностью автономна.

**Нет нужного UD-признака для моего языка?**
Создайте собственный в формате `Категория=Значение` (например, `Classifier=Yes`).

**Как добавить тексты в корпус?**
Дополните файл новыми предложениями (с последовательной нумерацией) и загрузите через Транслятор заново.
